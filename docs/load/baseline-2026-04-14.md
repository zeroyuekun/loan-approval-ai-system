# Baseline Load Test — 2026-04-14

**Config:** 50 concurrent users, 10 min steady (after 50s ramp), Locust 2.32.1 against `docker compose up` local stack. Traffic mix: 70% QuickScore (login + loans:create + ml:predict), 30% FullApplication (login + loans:create + agents:orchestrate + tasks:status polling).

**Throttle overrides for test:** `DJANGO_THROTTLE_USER_RATE`, `DJANGO_THROTTLE_PREDICT_RATE`, `DJANGO_THROTTLE_LOGIN_RATE`, `DJANGO_THROTTLE_ORCHESTRATE_RATE` all set to `10000/min` to measure capacity rather than throttling. Production defaults unchanged.

**Test user:** `loadtest` (promoted to `officer` role so it can poll task status while PENDING — see Observations).

## Observations and candidate bottlenecks

1. **auth:login p50 = 1300 ms** is the single biggest per-request latency in the baseline. Simple JWT + bcrypt password check should not take >1s. Likely causes: high bcrypt rounds configured in Django settings, or a slow per-request middleware (audit logging, correlation ID, etc.). **Top candidate for a Phase 2 fix.**
2. **loans:create p50 = 93 ms but p75 = 1000 ms** — bimodal distribution suggests a periodic synchronous blocker (e.g. a queryset that lacks `select_related`, or an audit-log write that occasionally contends). **Candidate for a Phase 2 fix.**
3. **tasks:status RemoteDisconnected (186 failures)** — looks like a WSGI worker reaping connections under load. Either the backend runs with too few workers/threads for the load or a keep-alive timeout is too short. **Candidate for a Phase 2 fix.**
4. **ml:predict p99 = 1100 ms** — tail latency is ~35x the median. Occasional cold re-load of the model artefact, GC pause, or lock contention on the active-model lookup. Worth investigating but lower priority since p95 is well inside target.
5. **celery_worker_io memory at 54% of limit** — not failing now, but climbs with every orchestration. Memory leak or large-payload caching candidate. Log for later.
6. **TaskStatusView ownership check returns 403 while PENDING for non-staff users** (`backend/config/urls.py:67`). This makes customers unable to poll their own task until it finishes, which breaks the documented 2-second polling pattern in CLAUDE.md. Worked around by promoting the load-test user to `officer`. Worth a bug filing.
7. **Hardcoded production throttles exist on five views** (`auth/login`, `auth/register`, `auth/refresh`, `ml/predict`, `agents/orchestrate`). This commit made four of them env-configurable. Two remain untouched (`register`, `refresh`) — extend if they become blockers.
